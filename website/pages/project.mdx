# Starting a Project

<Callout type="info"> 
A guide to include design-driven principles in your project from the very beginning.
</Callout>

## Start of the Project

### Gather Initial Requirements

- Draw up a loose initial prototype. 
- Agree on a platform (Web, CLI, etc.).

### Agree Working practices

- Regular review of the prototype (i.e. every two weeks/every month).
- Agree on shared language
    - Domain Terminology
    - RSE Terminology 

This is an iterative process.

## Stages of the Development Lifecycle

- Specify tasks
- Development
- Testing/QA
    - User Testing
    - Accessibility Testing
- Review
- Release
